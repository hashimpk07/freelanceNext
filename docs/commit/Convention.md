# Conventional Commit Types

Use the following formats for commit messages when using commitlint:

| Type     | Example                                |
| -------- | -------------------------------------- |
| feat     | `feat: add login button`               |
| fix      | `fix(api): incorrect token validation` |
| chore    | `chore: update dependencies`           |
| refactor | `refactor(ui): clean navbar component` |
| docs     | `docs: update readme`                  |
| style    | `style: format code`                   |
| test     | `test: add unit tests for auth`        |
